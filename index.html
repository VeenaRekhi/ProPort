TRIVIA GAME

Created a Trivia game using JavaScript for the logic and jQuery to manipulate HTML. For layout of this app used valid HTML and stylish CSS.

Crated a trivia game that shows only one question until the player answers it or their time runs out. If the player selects the correct answer, a screen will show congratulating them for choosing the right option. After a few seconds, displaying the next question -- achieving it without user input. The scenario is similar for wrong answers and time-outs. If the player runs out of time, alerting the player that time's up and displaying the correct answer. Waiting for few seconds, then showing the next question.

If the player chooses the wrong answer, telling the player they selected the wrong option and then display the correct answer. Wait a few seconds, then show the next question. On the final screen, showing the number of correct answers, incorrect answers, and an option to restart the game (without reloading the page).

RESPONSIVE PORTFOLIO

Created portfolio with a mobile-responsive layout.

By writing CSS famous Media Queries in the css file I created a Responsive-Portfolio. Used three @media screen tags, each with one of these max-widths: 980px, 768px and 640px.

I used 980px so that any of the content will not cut off. Since the desktop layout is about 960px wide, I wanted the media queries to kick in before any content gets cut off. 768px is about the width of a tablet and 640px is about the width of a phone in landscape.

LIRI- A NODE APP

LIRI is a Language Interpretation and Recognition Interface.

LIRI is like iPhone's SIRI. However, while SIRI is a Speech Interpretation and Recognition Interface, LIRI is a Language Interpretation and Recognition Interface. LIRI will be a command line node app that takes in parameters and gives you back data. LIRI will display your latest tweets. To retrieve the data for this app, i sent requests to the Twitter, Spotify and IMDB APIs, and installed respective Node packages . Twitter, Spotify, Request to grab data from the OMDB API.

This Liri app will take commands from the user and give respective output.

  • node liri.js my-tweets This will show your last 20 tweets and when they were created at in your terminal/bash window.
  • node liri.js spotify-this-song '' This will show the following information about the song in your terminal/bash window Artist(s) The song's name A preview link of the song from Spotify The album that the song is from if no song is provided then your program will default to "The Sign" by Ace of Base
  • node liri.js movie-this '' This will output the following information to terminal/bash window: * Title of the movie. * Year the movie came out. * IMDB Rating of the movie. * Country where the movie was produced. * Language of the movie. * Plot of the movie. * Actors in the movie. * Rotten Tomatoes Rating. * Rotten Tomatoes URL.
  • node liri.js do-what-it-says Using the fs Node package, LIRI will take the text inside of random.txt file and then use it to call one of LIRI's commands.

    • Feel free to change the text in that document to test out the feature for other commands.

    BURGER-SMASH

    Eat-Da-Burger! is a restaurant app with MySQL, Node, Express, Handlebars and ORM which follows the MVC design pattern!

    This app has a semantic templating framework, Handlebars and link with MySQL database to front-end applications. With all these created a burger logger for a fastfood restaurant with MySQL, Node, Express, Handlebars and a ORM. This app follows the MVC design pattern; using Node server and MySQL database to query and route data into the app, and Handlebars to generate it's HTML.

    Eat-Da-Burger! is a first stage fast-food restaurant app that lets users input the names of burgers they'd like to eat. Whenever a user submits a burger's name, this app will display the burger on the left side of the page -- waiting to be devoured. Each burger in the waiting area also has a Devour it! button. When the user clicks it, the burger will move to the right side of the page. This app will store every burger in a database, whether devoured or not.
